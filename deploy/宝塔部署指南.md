# 宝塔面板部署指南（简明版）

> 完整版见 [`baota.md`](./baota.md) · 上传清单见 [`BAOTA-UPLOAD.md`](./BAOTA-UPLOAD.md)

本文按「从零到上线」顺序说明，假设你已有 Linux 服务器并安装了宝塔面板。

---

## 一、准备

### 1. 域名与 DNS

| 用途 | 示例 | 说明 |
|------|------|------|
| 前端 | `www.你的域名.com` | 用户访问官网与控制台 |
| 后端 API | `api.你的域名.com` | Nginx 反代到本机 8000 端口 |

两条 **A 记录** 都指向服务器公网 IP。

### 2. 宝塔软件商店安装

- **Nginx**（最新稳定版）
- **PostgreSQL** 14+（推荐 16）
- **Redis** 7.x
- **Python 项目管理器** → 安装 **Python 3.11**
- **Node 版本管理器** → 安装 **Node 18** 并设为默认
- **Supervisor 管理器**

### 3. 创建数据库

宝塔 → **数据库** → PostgreSQL → 创建：

- 库名：`capability_network`
- 用户：`postgres`（或自定义，与 `.env` 一致）
- 密码：**强密码**（不要用 `postgres`）

若已安装 [pgvector](https://github.com/pgvector/pgvector)，在库中执行：

```sql
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```

未安装 pgvector 时，`.env` 可临时设 `LOCAL_SCHEMA=1` 应急；正式运营建议安装 pgvector 后改回 `LOCAL_SCHEMA=0`。

## 三、配置 `.env`

```bash
cd /www/wwwroot/capability-network
cp .env.example .env
ln -sf /www/wwwroot/capability-network/.env /www/wwwroot/capability-network/backend/.env
```

编辑 `.env`，**至少修改以下项**：

```bash
DEBUG=false
JWT_SECRET_KEY=<随机32位以上字符串>
DATABASE_URL=postgresql+asyncpg://postgres:你的强密码@127.0.0.1:5432/capability_network
CORS_ORIGINS=https://www.你的域名.com,https://你的域名.com
LOCAL_SCHEMA=0
VITE_API_URL=https://api.你的域名.com/api/v1
```

## 四、后端部署

```bash
cd /www/wwwroot/capability-network/backend

# 创建虚拟环境（Python 路径按宝塔实际安装调整）
/www/server/pyporject_evn/versions/3.11.9/bin/python3.11 -m venv .venv
source .venv/bin/activate

pip install -U pip
pip install -r requirements.txt

# 数据库迁移
alembic upgrade head
```

**上线前清理演示数据**（若本地曾跑过 seed / E2E）：

```bash
python scripts/purge_demo_data.py --dry-run    # 先预览
python scripts/purge_demo_data.py --confirm    # 确认删除 QA 账号
```

**Supervisor 守护进程**（宝塔 → Supervisor 管理器 → 添加）：

| 字段 | 值 |
|------|-----|
| 名称 | `capability-network-backend` |
| 运行目录 | `/www/wwwroot/capability-network/backend` |
| 启动命令 | `/www/wwwroot/capability-network/backend/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 2` |

保存并启动，本机验证：

```bash
curl -s http://127.0.0.1:8000/health
# 期望 {"status":"ok",...}
```

## 五、前端构建与静态站点

### 在服务器构建

```bash
cd /www/wwwroot/capability-network/frontend
export VITE_API_URL=https://api.你的域名.com/api/v1
npm ci
npm run build
```

### 或在 Windows 本地构建后上传 `dist/`

```powershell
cd frontend
copy .env.production.example .env.production
# 编辑 VITE_API_URL=https://api.你的域名.com/api/v1
npm ci
npm run build
# 将 dist/ 上传到服务器
```

宝塔 → **网站** → **添加站点**：

- 域名：`www.你的域名.com`
- 根目录：`/www/wwwroot/www.你的域名.com`
- 类型：纯静态

同步构建产物：

```bash
mkdir -p /www/wwwroot/www.你的域名.com
rsync -av --delete /www/wwwroot/capability-network/frontend/dist/ /www/wwwroot/www.你的域名.com/
```

站点配置中加入 SPA 路由（有 `try_files`）：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

## 六、Nginx 与 HTTPS

### 后端站点 `api.你的域名.com`

添加站点后，在配置中加入：

```nginx
location / {
    proxy_pass http://127.0.0.1:8000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 300s;
}
```

### SSL

两个站点都申请 **Let's Encrypt**，并开启 **强制 HTTPS**。

验证：

```bash
curl -sS https://api.你的域名.com/health
curl -sS -o /dev/null -w "%{http_code}\n" https://www.你的域名.com/
```

## 七、创建管理员

演示账号已清理，需注册真实管理员：

1. 浏览器打开 `https://www.你的域名.com/register` 注册账号
2. SSH 进服务器，将首个管理员提权：

```bash
cd /www/wwwroot/capability-network/backend
source .venv/bin/activate
python - <<'PY'
import asyncio
from sqlalchemy import select, update
from app.core.database import async_session
from app.auth.models import User

async def main():
    email = input("管理员邮箱: ").strip()
    async with async_session() as db:
        user = (await db.execute(select(User).where(User.email == email))).scalar_one_or_none()
        if not user:
            print("用户不存在"); return
        user.role = "admin"
        await db.commit()
        print(f"已将 {email} 设为 admin")

asyncio.run(main())
PY
```

3. 登录 `https://www.你的域名.com/admin`，在 **系统设置** 中修改站点名称、公告等。

## 九、常见问题

| 现象 | 处理 |
|------|------|
| `/health` 502 | Supervisor 是否 RUNNING；`ss -lntp \| grep 8000` |
| CORS 报错 | `.env` 的 `CORS_ORIGINS` 是否含 `https://www.你的域名.com` |
| `vector` 扩展错误 | 安装 pgvector 或临时 `LOCAL_SCHEMA=1` |
| 登录页还有演示账号 | 前端需用生产 env 重新 `npm run build` |
| Redis 未就绪 | 宝塔启动 Redis；`/health/ready` 需 Redis 正常 |
