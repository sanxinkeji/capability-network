## 0. 产品决策（统筹已定）

**问题**：MVP 交易链路完整，但演示路径像「发需求 + 人工接单」，与「AI 能力网络」定位脱节。

**决策**：**路线 A + 轻量 B** — 保留托管交易底座，新增 **AI 显性主路径**：

| 优先级 | 内容 | 会话 |
|--------|------|------|
| P0 | 一句话 AI 发需求、能力市场（智能体优先）、匹配推荐理由、MCP 接入引导 | **24** |
| P1 | 智能体供给 seed、Agent 自动交付演示、MCP deliver 补全 | **25** |
| P2 | 语义匹配 pgvector（生产） | 后续 |
| 暂缓 | 总后台运营增强、Git/宝塔 | 24–25 完成后再开 |

**标准演示路径（目标）**：首页「体验 AI 发单」→ 自然语言 → 解析 → 匹配（带理由）→ 选 **智能体供给** → 支付 → **自动交付**（25）→ 确认完成。

## 3. 设计原则（iOS 26）

详见 `docs/design-system-ios26.md`。摘要：

- **Liquid Glass**：半透明 + `backdrop-filter` 模糊
- **系统色**：Primary `#007AFF`，背景分层灰
- **圆角**：卡片 20px，按钮 pill
- **字体**：`-apple-system` / SF Pro 栈
- **信息架构**：官网（公开）与控制台（`/app`，需登录）分离

---

## 4. 合并与冲突规则

1. **先后端契约，后前端**：改 API 必须先更新 `docs/api-contract.md`
2. **样式只改 `frontend/src/styles` 与各 vue scoped**；禁止引入新 UI 框架（保持轻量）
3. **路由**：公开页无前缀；业务页统一 `/app/*`
4. **每会话结束**：`pytest`（若动后端）+ `npm run build`（若动前端）
